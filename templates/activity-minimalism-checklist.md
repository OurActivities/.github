# Activity Minimalism Checklist

Version: 1.0

---

# Purpose

Use this checklist before creating:

- activity
- repository
- workflow
- service
- package
- library

---

# Problem Definition

[ ] Clear problem statement exists

[ ] Human value identified

[ ] Activity outcome identified

---

# Existing Solution Review

[ ] Existing activity reviewed

[ ] Existing repository reviewed

[ ] Existing workflow reviewed

[ ] Existing automation reviewed

[ ] Existing template reviewed

---

# Simplicity Review

[ ] Can we do nothing?

[ ] Can documentation solve it?

[ ] Can configuration solve it?

[ ] Can workflow automation solve it?

[ ] Can an existing activity solve it?

---

# Repository Review

[ ] New repository justified

[ ] Repository scope is small

[ ] README exists

[ ] Examples exist

---

# Architecture Review

[ ] No premature abstraction

[ ] No speculative requirements

[ ] No unused extension points

[ ] No unnecessary framework

---

# Security Review

[ ] Authentication reviewed

[ ] Authorization reviewed

[ ] Validation reviewed

[ ] Privacy reviewed

[ ] Compliance reviewed

---

# Maintainability Review

[ ] Easy to understand

[ ] Easy to explain

[ ] Easy to test

[ ] Easy to modify

---

# Final Decision

If all checks pass:

Activity approved.

Otherwise:

Simplify further.
