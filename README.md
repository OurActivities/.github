.github

Organization-wide governance, standards, guidance, and community resources for the OurActivities ecosystem.

⸻

Purpose

This repository contains resources that apply across the entire OurActivities organization.

The goal is to establish a shared foundation for:

* Contributors
* Maintainers
* AI agents
* Automation systems
* Future activity repositories

This repository does not contain activity implementations.

Instead, it contains the guidance used to create and maintain them.

⸻

Philosophy

OurActivities is built around a simple idea:

Activities are the fundamental building blocks of human and organizational behavior.

Everything within this organization should support the understanding, execution, improvement, and sharing of activities.

⸻

Guiding Principles

Activity First

Activities come before tools.

Activities come before software.

Activities come before automation.

Understand the activity before designing a solution.

⸻

Simplicity First

OurActivities follows a Ponytail-inspired architectural philosophy.

Before creating:

* repositories
* services
* frameworks
* workflows
* libraries

we ask:

1. Does this need to exist?
2. Can an existing activity solve it?
3. Can documentation solve it?
4. Can configuration solve it?
5. Can automation solve it?
6. Only then should we write code.

⸻

Documentation First

Documentation is a first-class deliverable.

Every activity should be understandable before it is automated.

⸻

Human and AI Collaboration

Activities may be performed by:

* Humans
* AI agents
* Software systems
* Hybrid teams

The objective is collaboration rather than replacement.

Contribution Guidelines

When contributing:

1. Prefer simplicity.
2. Prefer reuse.
3. Prefer documentation.
4. Avoid premature abstraction.
5. Keep repositories focused.
6. Keep activities understandable.

If a solution feels overly complex, simplify it.

⸻

Success Criteria

The ecosystem is successful when activities are:

* visible
* understandable
* reusable
* improvable
* enjoyable

not when they contain the most code.
