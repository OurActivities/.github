OurActivities

Activities are the building blocks of life.

Welcome to OurActivities.

OurActivities is an open ecosystem for defining, documenting, automating, measuring, improving, and sharing activities.

An activity can be as simple as:

* Waking up
* Showering
* Having breakfast
* Commuting
* Sprint planning

Or as complex as:

* Running a project
* Organizing an event
* Building software
* Managing a department
* Operating a business

OurActivities treats every activity as something that can be:

* Described
* Learned
* Practiced
* Improved
* Shared
* Automated

⸻

Vision

We believe that most systems focus on applications, projects, or organizations.

We focus on activities.

Activities exist everywhere:

* At home
* At school
* At work
* In communities
* In governments
* In software systems
* In AI systems

By understanding activities, we can better understand how people and systems create value.

⸻

Principles

Activity First

Activities come before tools.

Activities come before software.

Activities come before automation.

We first understand the activity, then determine how technology can support it.

⸻

Simplicity Before Complexity

OurActivities follows a minimalistic design philosophy inspired by Ponytail.

Before creating:

* A new repository
* A new workflow
* A new service
* A new package
* A new framework

We ask:

1. Does this need to exist?
2. Can an existing activity solve it?
3. Can documentation solve it?
4. Can configuration solve it?
5. Can automation solve it?
6. Only then should we write code.

⸻

Documentation First

Documentation is a first-class deliverable.

Every activity should be understandable by humans before it is automated by machines.

⸻

Human and AI Collaboration

Activities can be performed by:

* Humans
* AI agents
* Software systems
* Hybrid teams

OurActivities explores how humans and AI can collaborate effectively while keeping humans in control of outcomes.
